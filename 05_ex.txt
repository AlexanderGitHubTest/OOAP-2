Использование 5 принципов повторного использования модуля в python.
1. Обобщённые типы или типы-генерики. На уровне модуля нет понятия "тип модуля", 
поэтому данный принцип на уровне модуля не поддерживается, он поддерживается только для объектов внутри модуля. 
Объекты внутри модуля имеют динамическую типизацию, но возможно указать тип, используя модуль typing 
(как генерик, так и какое-либо сочетание типов), а проверку сделать type-чекером, типа mypy.
2. Объединять несколько функций. Python это поддерживает базово. Функции внутри модуля свободно обращаются друг к другу. 
Есть механизм импорта в модуль ("import <имя модуля>", "from <имя модуля> import <сущность 1, ... сущность n>",
"from <имя модуля> import *" ), который позволяет использовать в одном модуле функции из нескольких других модулей, при этом не копируя сам код.
3. Семейство модулей. Поддерживается. Механизм пакетов, позволяющий поместить модули для решения общей задачи в одну папку и, 
используя __init__.py, ограничить (через __all__) и  указать (через импорт модулей пакета в __init__.py) публичное API семейства.
4. Реализацию родительского модуля динамически (полиморфно). На уровне модуля не получится создать один модуль на основе другого. 
Такое возможно только на уровне объектов внутри модуля. Как-то в ручном варианте можно частично реализовать, например, 
через namespaced packages и importlib, но это уже программное решение, не уровня поддержки python.
5. Интегрировать общее поведение нескольких модулей. Базово не реализовано. Можно сделать частично 
через использование __getattr__ внутри __init__.py для подстановки нужного импорта, но это очень "ручное" решение.

Итого поддерживаются 2 и 3. 1 не поддерживается. 4 и 5 как-то частично программно только если можно сделать.
