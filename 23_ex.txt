Приведите словесный пример иерархии классов, и объясните, почему в ней нельзя ослаблять постусловия и усиливать предусловия.

Рассмотрю иерархию Set -> SortedSet (Множество -> Отсортированное множество).


1. Постусловия и предусловия.

1.1. Какие изменения постусловий и предусловий произошли:

- команда add
-- было постусловие "в множество добавлено новое значение"
-- стало постусловие "в множество добавлено новое значение
                      и значения в множестве отсортированы по возрастанию"
Это усиление первичного постусловия. С точки зрения родительского класса главное, 
что значения сохранены. Добавление сортировки усиливает постусловие.

- команда remove
-- было предусловие "в множестве имеется значение value"
-- стало предусловие "нет предусловий"
Это ослабление первичного предусловия. То что разрешено спецификацией родительского класса, разрешает и наследник.

1.2. Почему нельзя ослаблять постусловия. Представим, что хотим сделать не SortedSet, 
а SortedEvenSet (отсортированное множество, где будут только четные значения).
В команду add придётся написать усиленное предусловие "value только чётное", 
либо ослабленное постусловие "сохраняем только если value чётное".
В обоих случаях будут уже варианты, когда нельзя будет подставить SortedEvenSet вместо Set.
Для нечётных value SortedEvenSet не отработает так же, как Set:
- в случае усиленного предусловия, мы не соблюдаем спецификацию исходного класса 
и запрещаем допустимые в исходном классе значения;
- в случае ослабленного постусловия мы получим некорректные состояния (нам значения передали, а мы их "потеряли").


2. АТД для Set и SortedSet

// Множество неограниченного размера
АТД Set

abstract class Set<T>

  // конструктор
// постусловие: создано пустое множество
  public Set<T> Set();

  // команды
// предусловие: нет
// постусловие: в множество добавлено новое значение
  public void add(T value); 

// предусловие: в множестве имеется значение value
// постусловие: из множества удалено значение value
  public void remove(T value); 

  // запросы
// предусловие: нет
  public bool contains(T value); // содержится ли значение value в таблице

// предусловие: нет
  public Sequence<T> elements(); // вернуть элементы множества как последовательность
                                 // в произвольном порядке


// Отсортированное множество неограниченного размера
АТД SortedSet

abstract class SortedSet<T> : Set<T>

  // конструктор
// постусловие: создано пустое множество
  public SortedSet<T> SortedSet();

  // команды
// предусловие: нет
// постусловие: в множество добавлено новое значение
// и значения в множестве отсортированы по возрастанию
  public void add(T value); 

// предусловие: нет
// постусловие: из множества удалено значение value,
// если значения value в множестве не было, то ничего не изменено
  public void remove(T value); 

  // запросы
// предусловие: нет
  public bool contains(T value); // содержится ли значение value в таблице

// предусловие: нет
  public Sequence<T> elements(); // вернуть элементы множества как последовательность
                                 // по возрастанию
