Рассматриваю python.
1. Управлением разрешением импорта объектов. Первые два варианта ограничивают список для массового импорта 
(from <имя модуля> import * ), 3-й вариант универсальный (как для массовго, так и для штучного импорта). 
Вариант 1 - использовать глобальную переменную __all__ и указать список разрешённых к импорту объектов; 
по сути список в __all__ является публичным интерфейсом модуля. 
Вариант 2 - запрещенных для импорта объектам начинать имя с "подчёркивания". 
Вариант 3 - воспользоваться различными областями видимости переменных и убрать объекты (переменные, функции, классы), 
которые нельзя отдельно вызывать внутрь классов или функций пространства имён модуля (которые можно импортировать напрямую).
2. Управление экспортом объектов из другого модуля. Механизм импорта в python позволяет взять 
только только необходимые объекты из другого модуля, либо все объекты. Также можно отделить пространство имён 
другого модуля (import <имя модуля>), либо импортировать в пространство имён текущего модуля (from <имя модуля> import <объект1,.. объект n>).
